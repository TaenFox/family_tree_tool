# Family Tree Tools

Локальный GUI для ведения семейной картотеки.

Состав каталога:

- `server/` — пакет локального HTTP-сервера и API (модули по доменам);
- `static/` — HTML, CSS и JavaScript интерфейса;
- `template-data/docs/` — шаблон картотеки без личных данных;
- `config.example.json` — пример конфигурации;
- `config.json` — рабочая конфигурация, в git не хранится.

Подготовка:

```bash
cp config.example.json config.json
pipenv install
```

Пример конфигурации по умолчанию использует `./workspace`.

Если каталога `workspace/docs` ещё нет, сервер при первом запуске создаст его из `template-data/docs`.

Если данные хранятся в отдельном репозитории, в `config.json` можно указать внешний `workspace_dir`. Это должен быть корень рабочего каталога, внутри которого находится `docs/`.

Запуск из корня проекта:

```bash
pipenv run python -m server
```

Сервер поднимается на FastAPI/uvicorn (ASGI).

Интерфейс открывается по адресу `http://127.0.0.1:8765`.

Если `config.json` отсутствует, инструмент работает в режиме совместимости и пытается использовать `../docs`. Это нужно только для текущего репозитория, пока инструмент ещё не вынесен отдельно.

GUI работает с карточками в `docs/`:

- люди — `03-people/`;
- группы — `04-groups/`;
- места — `05-places/`;
- источники — `06-sources/`;
- исследования — `07-research/`.

Что умеет:

- создавать и редактировать карточки;
- подставлять следующий номер;
- пересобирать индексы разделов и реестр;
- вставлять ссылки на существующие карточки;
- синхронизировать встречные связи людей и групп;
- показывать общий граф людей и граф отдельной карточки;
- работать с фотографиями карточки через `images/`.

Фотографии:

- загрузка drag-and-drop, выбором файла или из буфера обмена;
- хранение в `images/` карточки;
- выбор главного фото из загруженных изображений;
- превью, миниатюры, удаление и полноэкранный просмотр.

Ограничения:

- номер и тип существующей карточки не меняются;
- связи источников и мест не синхронизируются автоматически в обратную сторону;
- отдельного раздела фотографий в `card.adoc` нет: GUI управляет только каталогом `images/` и главным фото.

API внутренний, используется самим GUI.
